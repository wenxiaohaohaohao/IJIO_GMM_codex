## 权限规则（当前有效）

本文档是当前工作区权限规则的唯一有效入口。

### 只读可调用目录

以下目录及其全部子目录/文件为“只读、可调用”：

- `reference_materials/`
- `备份/`

约束：

- 允许：读取、引用、在代码中调用
- 禁止：修改、覆盖（replace）、重命名、移动、删除

### 可删改替换目录

除 `reference_materials/` 与 `备份/` 之外，工作区其余目录/文件统一视为“可操作”：

- 允许：新增、修改、replace（覆盖写入）、重命名、移动、删除

### 执行口径

- 任何自动化脚本或人工操作，均不得对 `reference_materials/`、`备份/` 执行写操作
- 运行代码时，如果需要输入文件，可从只读目录读取；输出文件应写入非只读目录
- 清理临时文件、重跑结果、替换中间产物，均应在可操作目录中完成

### 属性判定规则（Codex 执行时使用）

- 标记为 `只读可调用`：只允许读取和调用，不允许修改、replace、删除、重命名、移动
- 标记为 `可删改替换`：允许新增、修改、replace、删除、重命名、移动
- 标记为 `规则文档/说明文档，可编辑`：按普通可编辑文件处理

## 快速开始

1. 使用 PowerShell 进入仓库根目录：`D:\paper\IJIO_GMM_codex_en`
2. 先进入任务对应目录（例如 `1017/1022_non_hicks/code/`），再运行对应脚本
3. 将输出写入非只读目录
4. 变更后用 `git status` 检查工作区状态

## 维护建议

- 临时脚本统一使用 `tmp_` 前缀，任务完成后及时清理
- 结果文件尽量写入固定结果目录，避免散落在根目录
- 提交前优先检查是否误改只读目录

## 日常开发执行约束（Default permission）

1. 永远在正确的 workspace/repo 里运行任务。
2. 先 Review 再 Apply，先读后改。
3. 改动保持最小化，小步提交，保证可回滚。
4. 不运行重计算、GUI、或高风险命令，除非你明确要求。

Hard constraint:
- 仅允许修改当前 workspace 内文件；不得触碰 workspace 之外的任何文件。
